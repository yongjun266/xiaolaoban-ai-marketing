# 小老板 AI 营销助手开发路线图

本路线图以 `docs/PRD.md` 为唯一产品需求依据。各阶段采用小步迭代方式开发，每一步都应保持项目可以运行。

## 第一阶段：V1（MVP）

目标：验证 AI 文案生成能力，完成一个无需登录、可部署、可演示的 AI Web 应用。

- Next.js 初始化
- 首页（Landing Page）
- AI 工作台
- 商品信息输入
- AI 商品标题生成
- AI 商品介绍生成
- AI 营销文案生成
- AI 客服回复生成
- 一键复制
- 加载状态
- 错误提示
- OpenAI API 接入
- 响应式页面
- Vercel 部署

V1 不开发用户登录、Supabase、数据库、历史记录和收藏功能。

## 第二阶段：V2

目标：在核心 MVP 稳定后，增加用户体系和内容保存能力。

- 用户登录（Supabase Auth）
- Supabase PostgreSQL 数据库
- 历史记录
- 收藏功能
- 重新生成
- 用户配置

## 第三阶段：V3

目标：扩展内容形态和商业化能力。

- AI 海报图片生成
- 模板管理
- Prompt 模板库
- 多模型支持
- Token 消耗统计
- 会员系统

如果开发任务超出当前版本范围，应先提醒用户并等待确认，不得擅自提前实现。
